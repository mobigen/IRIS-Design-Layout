Check Box

체크 박스는 목록 중 선택할 항목이 여러개 있을때 사용합니다.

  • 체크박스는 2개 크기로 구분합니다.
  • 체크박스는 default, hover, checked, indeterminate, disabled 총 5개 상태를 갖습니다.
  • 체크박스를 하나만 사용하는 경우 toggle-switch를 사용해 두 상태를 전환하는 것과 같습니다.
    두개의 차이는 Switch는 상태 변경이 직접 반영되며, Checkbox는 상태 변경 후 Submit 동작을 수행해야 반영됩니다.

Loader

로더는 데이터를 검색하거나 화면이 느리게 뜨는 경우 사용합니다.
사용자에게 프로세스가 진행중임을 알리는데에 도움이 됩니다.

  • 로더는 스피너 형태와 바 형태 2개로 구분합니다.
    • 스피너형 로더
      • 로더는 3개 사이즈로 구분합니다.
        • loader--sm: 버튼 등 Form 요소나 텍스트 수준 영역 로딩
        • loader--md: 카드 수준의 블록 영역 로딩
        • loader--sm: 페이지 영역 로딩
      • overlay는 loader 아래에 있는 컨텐츠를 조작할 수 없는 경우 사용합니다.
      • 로더는 오버레이는 3개 스타일로 구분합니다.
        • overlay--dark
        • overlay--mono
    • 바형 로더
      • 바형 로더는 2개 스타일로 구분합니다

Notification

노티피케이션은 사용자에게 정보를 전달하는 알림 메시지입니다.

  • 노티피케이션은 화면 일부를 가리며 나타나는 일반적인 유형과 메시지만 표시해주는 두가지 유형으로 구분합니다
  • Message 유형은 Validation에 사용합니다. (구, mui-message)
  • Message 유형은 notification-message--inline 클래스를 추가하여 inline-flex로 사용할 수 있습니다
  • 노티피케이션은 총 4가지 상태가 있고 의미는 다음과 같습니다.
    • info : 일반적인 정보전달이나 메세지 역할을 하는 알림에서 사용됩니다.
    • success: 사용자의 동작이 성공적으로 이루어졌을 때 사용됩니다. (ex 등록완료)
    • warning: 사용자가 단계를 수행하기 앞서, 제약이 따를만한 상황을 경고할 때 사용됩니다. 이를 감안하고 사용자는 동작을 강행할 수 있습니다.
    • error: 치명적인 오류가 발생할 때 사용됩니다. 이 상태에서 사용자는 다음 단계를 수행할 수 없습니다.

Select

특정 버튼을 클릭했을 경우 뜨는 리스트 형태의 메뉴입니다.

  • 셀렉트의 유형이 2개 있습니다.
    • 텍스트 / 아이콘 + 텍스트 / 텍스트 + 서브텍스트
    • 멀티 셀렉트 - 다중 선택이 가능한 UI
    • 콤보 박스 - 입력 필드와 리스트가 결합한 UI
  • 셀렉트의 테마가 2개 있습니다.
    • select - 보더가 있는 UI
    • select--borderless - 보더가 없는 UI
  • 셀렉트의 사이즈가 3개 있습니다.
    • select--lg / select / select--sm
  • 셀렉트의 리스트의 방향을 설정할 수 있습니다.
    • 기본 - 셀렉트 버튼을 기준으로 좌측 하단으로 펼쳐지는 UI
    • select--left - 셀렉트 버튼을 기준으로 좌측에서부터 셀렉트 리스트가 나열되는 UI
    • select--up - 셀렉트 버튼을 기준 위로 셀렉트 리스트가 펼쳐지는 UI
    • select--right - 셀렉트 버튼을 기준으로 우측에서부터 셀렉트 리스트가 나열되는 UI
  • select--view-all 클래스를 추가하면 셀렉트 리스트가 텍스트 길이에 따라 계속해서 늘어납니다.
